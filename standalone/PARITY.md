# Standalone validator parity

The JavaScript implementation is an independently maintained partial port of
the BBC Python validator at commit
`5955c71bbba9ceb7bde39f19730c925a0ce1e9c7`.

`rule-manifest.json` is the machine-readable record of claimed rule codes,
partial-code limitations, the counting model and the pinned validator baseline.

The standalone report shall count both:

- **issue types**: an exact severity, code and message tuple;
- **occurrences**: every affected source location belonging to that tuple.

Overall validity summary records are not independent issue types and are not
generated by the standalone implementation.

Run a parity audit after producing a standalone report from the same TTML:

```bash
python scripts/compare_standalone_parity.py \
  --ttml example.xml \
  --standalone-report example.xml.validation-report.html \
  --presentation horizontal \
  --validator-command "uv run validate-ttml"
```

The audit runs the official validator, expands its collated `N locations`
records, and fails if any claimed JavaScript rule differs in severity, code,
message or occurrence count. A rule shall not be added to the claimed set until
it has positive, negative and repeated-occurrence fixtures.

Passing this audit establishes parity only for the explicitly claimed rules. It
does not establish full EBU-TT-D, TTML, IMSC or BBC validator parity.
